Codebook

Complete documentation for every field in the SSWR Conference History Database. The dataset consists of two relational tables linked by paper identifier. For methodology category definitions, see the Methodology page.

Relational structure: The papers table contains one row per presentation. The paper_authors table contains one row per author–presentation association. Join on papers.id = paper_authors.paper_id. A single presentation may have multiple author records; a single researcher may appear across multiple presentations.

Papers Table

23,793 records · 8 fields · One row per presentation

id String

Unique identifier for each presentation.

Format: YYYY-L-NNNN where YYYY is the conference year, L is a letter code indicating the source format, and NNNN is a sequential number.

Example: 2019-P-0142 Missing: 0% Unique: 23,793
year Integer

Conference year in which the presentation was given. Ranges from the 9th Annual Conference (2005) through the 30th Annual Conference (2026).

Range: 2005–2026 Missing: 0%
title String

Presentation title as listed in the Confex conference program.

Missing: 0%
abstract String

Full abstract text as submitted by the authors to the Confex system. May contain minor encoding artifacts from the original HTML source.

Missing: 0%
author_count Integer

Number of authors listed on the presentation. Derived from the count of associated records in the paper_authors table.

Range: 1–20+ Mean: 2.94 Missing: 0%

Final consensus methodology classification for the presentation, based on abstract content. This is the primary methodology field recommended for analysis. See the Methodology page for full category definitions and the classification process.

Paper Embeddings Table

23,793 records · One row per presentation · Available via API only

Important: Two embedding models are provided. Your query embedding must use the exact same model as the column you search against. Both are available via OpenRouter or OpenAI. Using a different model produces meaningless results.

paper_id String · Foreign Key

Foreign key linking to papers.id.

Unique: 23,793
embedding_large Vector (3,072 dimensions)

Dense vector from OpenAI’s text-embedding-3-large. Higher quality, best for precision-critical similarity search and clustering.

Model: text-embedding-3-large Dims: 3,072 RPC: match_papers(query_embedding, ...)
embedding_small Vector (1,536 dimensions)

Dense vector from OpenAI’s text-embedding-3-small. Faster and cheaper to query, suitable for most use cases.

Model: text-embedding-3-small Dims: 1,536 RPC: match_papers_small(query_embedding, ...)
created_at Timestamp

Timestamp of when the embeddings were generated.

Embeddings are available through the Supabase REST API and MCP connection but are not included in the downloadable CSV/JSON files due to their size.

Paper Authors Table

69,924 records · 19 fields · One row per author–presentation association

Temporal note: All author information (position, degree, institution, affiliation) reflects what was reported at the time of the conference, not current information. This database preserves the original SSWR conference record and is not a directory of current positions or affiliations. A researcher listed as a doctoral student in 2012 may now hold a faculty position — this is not an error.

id Integer

Unique identifier for each author–presentation record.

Missing: 0% Unique: 69,924
paper_id String · Foreign Key

Foreign key linking to papers.id. Identifies the presentation this author record is associated with.

Missing: 0%
author_order Integer

Position of the author in the author list. A value of 1 indicates the first (presenting) author.

Range: 1–20+ Missing: 0%
name String

Author name as it appeared in the Confex system, preserving original casing and formatting.

Example: Brian E. Perron Missing: 0%
name_normalized String

Lowercased version of the author name with NFKD Unicode normalization applied. Accented characters are converted to ASCII equivalents (e.g., “é” → “e”), multiple spaces collapsed, and leading/trailing whitespace removed. Used for matching and deduplication.

Missing: 0%
degree String

Academic degree(s) or credentials extracted from the author’s affiliation string by the language model.

Examples: PhD, MSW, PhD, LCSW Missing: 10.7% (7,490 records)
position String

Raw position or job title as extracted from the Confex affiliation text by the language model. This is the unprocessed text before normalization.

Examples: Assistant Professor, PhD Student, Associate Dean for Research Missing: ~8%
position_normalized String · Categorical

Standardized position category assigned through keyword-based classification of the raw position field. See the Methodology page for the full taxonomy and classification rules.

institution String

Cleaned institution name extracted by the language model. Represents the primary academic or organizational affiliation. Authors listing multiple affiliations were assigned to the first-listed institution.

Example: University of Michigan Missing: <1%

Original institutional affiliation text as it appeared in the Confex system, before any processing or normalization.

Example: School of Social Work, University of Michigan, Ann Arbor, MI
institution_normalized String

Standardized institution name from a canonical mapping table. Variants such as “University of Michigan,” “U. Michigan,” “U-M,” and “UM School of Social Work” are consolidated to a single canonical form. Institutions without a canonical mapping retain their cleaned form.

Unique values: 4,049 Missing: 0.3% (229 records)

Normalized institution identifier linking to the institution mapping table.

Missing: ~2%

City of the author’s institution, as extracted from the affiliation text.

Example: Ann Arbor
state_province String

State or province of the author’s institution. U.S. state abbreviations and Canadian province abbreviations were expanded to full names (e.g., “MI” → “Michigan,” “ON” → “Ontario”).

Example: Michigan
country String

Country/region as extracted from the affiliation text or inferred from state/city information (e.g., “Ann Arbor, MI” mapped to USA).

Missing: ~1%
country_normalized String

Standardized country/region name using a mapping dictionary of over 80 variations. Examples: “United States of America,” “US,” “U.S.,” “America” all mapped to “USA”; “Republic of Korea” and “Korea” mapped to “South Korea.”

Unique values: 93 Missing: 1.0% (697 records)
parsing_error Boolean

Flag indicating whether the language model encountered errors while parsing the affiliation string for this author record.

Values: true, false, or null
author_id Integer

Unique author identifier assigned during entity resolution. Each distinct name variant receives its own ID. Multiple records with the same author_id represent the same name string appearing across different presentations.

Unique values: 23,481
canonical_author_id Integer

Canonical author identifier after entity resolution and deduplication. Name variants determined to refer to the same individual (e.g., “Michael S. Sherraden,” “Mike Sherraden,” “M. Sherraden”) share the same canonical_author_id. Entity resolution reduced 23,481 raw name variants to 20,779 canonical identities.

Unique values: 20,779
