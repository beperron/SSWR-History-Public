MCP & Skill File

The Model Context Protocol (MCP) lets AI assistants like Claude query this database directly through conversation. No SQL, no downloads, no setup beyond a one-time configuration. You ask a question in plain language. The AI writes the query, runs it, and interprets the results.

This is an early example of how structured research datasets can become AI-accessible — collapsing the distance between a question and its answer.

What is MCP?

MCP is an open protocol that connects AI assistants to external data sources. When you connect Claude to the SSWR database via MCP, Claude can:

  • Run SQL queries against the live database
  • Inspect table schemas and relationships
  • Combine data across tables to answer complex questions
  • Return interpreted, contextualized results — not raw data dumps

The connection is read-only. Claude cannot modify, insert, or delete any records. This is enforced at both the MCP configuration level and the database level through row-level security policies.

Setup

Prerequisites

  • Node.js version 18 or later
  • Claude Desktop, Claude Code, or any MCP-compatible client

Step 1 — Open your configuration file

ApplicationFile
Claude Desktop (macOS)~/Library/Application Support/Claude/claude_desktop_config.json
Claude Desktop (Windows)%APPDATA%\Claude\claude_desktop_config.json
Claude Code~/.claude/settings.json

Create the file if it doesn’t exist.

Step 2 — Add the SSWR connection

Paste the following into your configuration file. If you already have other MCP servers, add sswr-history inside the existing mcpServers object.

{
  "mcpServers": {
    "sswr-history": {
      "command": "npx",
      "args": [
        "-y",
        "@anthropic-ai/mcp-supabase@latest",
        "--supabase-url",
        "https://jomsksqqcpkbuhwytovo.supabase.co",
        "--supabase-key",
        "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImpvbXNrc3FxY3BrYnVod3l0b3ZvIiwicm9sZSI6ImFub24iLCJpYXQiOjE3NDQyMDgwODksImV4cCI6MjA1OTc4NDA4OX0.9Vxp6gB03H0MKxrX1ltSCKCumlT-ba5GszWqAAw1Aqg",

Step 3 — Restart and verify

Restart the application. In Claude Desktop, the MCP server icon (hammer) should appear at the bottom of the chat window. Verify by asking:

“List the tables in the SSWR database and tell me how many rows each has.”

Example Prompts

Once connected, try these to get a feel for what’s possible:

Trends

  • “Show me the number of presentations per year from 2005 to 2026”
  • “How has the proportion of qualitative research changed over time?”
  • “What is the year-over-year growth rate in mixed methods presentations?”

Institutions & Geography

  • “Which 20 institutions have produced the most first-authored presentations?”
  • “How has international participation changed since 2010?”
  • “What countries are represented in the 2026 conference?”

Authorship & Careers

  • “What percentage of first authors are doctoral students vs. assistant professors?”
  • “How has the average number of authors per presentation changed?”
  • “Who are the 10 most prolific presenters across all years?”

Content & Methods

  • “Find presentations about child welfare from the last 3 years”
  • “What proportion of review papers are meta-analyses vs. scoping reviews?”
  • “Search for presentations mentioning ‘machine learning’ in the abstract”

Skill File

The repository includes a CLAUDE.md file — a structured reference that AI assistants automatically read when working within the repository. It contains:

  • Complete schema reference with types and descriptions
  • All categorical values with counts
  • Common SQL query patterns
  • Key relationships and join logic
  • Data quality notes and known limitations
  • OpenRouter embedding setup

Using the Skill File Locally

If you’re working with the CSV files in Claude Code rather than through MCP, clone the repository. The CLAUDE.md file is read automatically and gives Claude full context about the dataset structure.

git clone https://github.com/beperron/SSWR-History.git
cd SSWR-History
# Open Claude Code in this directory — it reads CLAUDE.md automatically

Using the Skill File with MCP

When connected via MCP, Claude can discover the schema by inspecting the database directly. The CLAUDE.md file adds context that isn’t in the schema — categorical value meanings, data quality caveats, recommended join patterns, and the citation requirement.

You can paste the contents of CLAUDE.md into your first message, or ask Claude to read it:

“Read the CLAUDE.md file from the SSWR-History repository and use it as context for our analysis.”

How It Works

When you ask a question, the following happens behind the scenes:

  1. You ask a question in natural language
  2. Claude inspects the database schema (tables, columns, types)
  3. Claude writes a SQL query that answers your question
  4. The MCP server executes the query against the Supabase database
  5. Claude interprets the results and responds in plain language

You can ask Claude to show the SQL it ran, request modifications, or chain follow-up questions. The entire conversation has access to the live data.

Security: The connection uses a read-only API key with row-level security. Claude can query papers, paper_authors, and paper_embeddings. It cannot access administrative tables, and it cannot modify any data.

Why This Matters

Most research datasets sit in repositories as downloadable files. Accessing them requires downloading, cleaning, loading into a statistical package, and writing analysis code — before you can ask your first question.

MCP inverts this. The dataset is live, queryable, and conversational. A doctoral student exploring SSWR trends, a faculty member preparing a conference submission, or a program officer reviewing the field’s research portfolio can all start with a question rather than a download.

We see this as a model for how scholarly datasets should be distributed — not just as files, but as living, queryable infrastructure that meets researchers where they already work.
