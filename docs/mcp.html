Setup Guide

Setup Steps

1

Add the config to your MCP settings file. No API key or Node.js required.

{
  "mcpServers": {
    "sswr-history": {
      "url": "https://mcp.supabase.com/mcp?project_ref=jomsksqqcpkbuhwytovo&read_only=true&features=database"
    }
  }
}
2

Restart the application. The SSWR database will be available immediately.

3

Try asking:

“How many presentations were at SSWR in 2026?”

What You Can Ask

Trends

“Show me the number of presentations per year from 2005 to 2026”
“How has the proportion of qualitative research changed over time?”

Institutions & Geography

“Which 20 institutions have the most first-authored presentations?”
“What countries are represented in the 2026 conference?”

Authorship

“What percentage of first authors are doctoral students?”
“Who are the 10 most prolific presenters across all years?”

Content

“Find presentations about child welfare from the last 3 years”
“Search for presentations mentioning ‘machine learning’ in the abstract”

Skill File

The repository includes a CLAUDE.md file that AI assistants read automatically. It contains the complete schema, common query patterns, data quality notes, and categorical value definitions. When working locally with Claude Code, clone the repository:

git clone https://github.com/beperron/SSWR-History-Public.git
cd SSWR-History-Public

Claude Code reads the skill file automatically and gains full context about the dataset.

How It Works

You ask a question. The AI inspects the database schema, writes SQL, executes it against the live Supabase database, and returns interpreted results. The connection is read-only — enforced at both the MCP configuration level and the database level through row-level security. The AI can query papers, paper_authors, and paper_embeddings but cannot modify any data.

Why MCP

Most research datasets require downloading files, cleaning data, loading into a statistical package, and writing analysis code before you can ask your first question. MCP inverts this. The dataset is live, queryable, and conversational. A doctoral student exploring trends, a faculty member preparing a submission, or a program officer reviewing the field can start with a question rather than a download.

We see this as a model for how scholarly datasets should be distributed — not just as files, but as queryable infrastructure that meets researchers where they already work.
