Quick Start

Why AI-Ready?

We designed this data release with AI in mind from the start. Rather than simply posting CSV files and a codebook, we carefully considered how researchers will actually work with data going forward — through AI assistants that can query, analyze, and interpret on their behalf.

AI-ready means the data ships with everything an AI assistant needs to work with it effectively: live database connections (MCP), modular skill files that teach the AI the schema and query patterns, and vector embeddings for semantic search. It works across Claude, Cursor, OpenCode, and other platforms.

This is not about handing off analysis. It’s about eliminating the hours spent on routine plumbing — loading files, writing SQL, debugging syntax — so researchers can invest their time where it matters: formulating questions, reviewing the generated code for correctness, and interpreting what the results mean. The scholarly contribution is never the code. It’s the question you ask and the interpretation you provide.

Step 1 — Connect to the Database

MCP (Model Context Protocol) lets your AI assistant query the live SSWR database directly. It works with Claude Desktop, Claude Code, Cursor, OpenCode, and other MCP-compatible tools.

Copy this prompt and paste it into your AI assistant:

Preview what gets copied

I want to connect to the SSWR Conference History Database using MCP (Model Context Protocol). This is a read-only research database with 23,793 conference presentations (2005–2026). No login required — it uses a public read-only key.

Please add an MCP server called “sswr-history” using the npx command with @supabase/mcp-server-supabase. The Supabase URL is https://jomsksqqcpkbuhwytovo.supabase.co and the read-only anon key is included in the prompt.

Steps:

  1. Make sure Node.js (version 18+) is installed
  2. Find my MCP configuration file
  3. Add the sswr-history server with the npx command, Supabase URL, anon key, and --read-only flag
  4. Help me restart the application
  5. Verify the connection by listing the available tables

Already know MCP? The raw JSON config is on the Download page.

Step 2 — Install the Skills

Skills are modular instruction files that give the AI specialized knowledge about this database — what the fields mean, how to write correct queries, how to run semantic search, and how to cite the data. Without them, the AI can still connect but won’t know the schema details or best query patterns.

Choose your tool and copy the prompt:

Claude Code reads skills from .claude/skills/ automatically.

Preview
Download and install the SSWR skills. Save each file to the specified path:
1. .claude/skills/sswr-codebook/skill.md (from github.com/beperron/SSWR-History-Public)
2. .claude/skills/sswr-query/skill.md
3. .claude/skills/sswr-semantic-search/skill.md
4. .claude/skills/sswr-about/skill.md
5. CLAUDE.md in project root
Confirm installed and list available tables.

Cursor reads skills from .cursor/rules/. The MCP config at .cursor/mcp.json is auto-discovered.

Preview
Download the SSWR database context files for Cursor. Save each to the project:
1. Download CODEBOOK_LLM.md from github.com/beperron/SSWR-History-Public and save to .cursor/rules/sswr-codebook.md
2. Create .cursor/mcp.json with the npx-based MCP config (command: npx @supabase/mcp-server-supabase, with Supabase URL and anon key). No login required.
Confirm the database is connected and list available tables.

OpenCode reads AGENTS.md automatically and uses opencode.jsonc for MCP config.

Preview
Download the SSWR database context for OpenCode:
1. Download AGENTS.md from github.com/beperron/SSWR-History-Public and save to AGENTS.md in the project root
2. Download opencode.jsonc from the same repo and save to opencode.jsonc
The AGENTS.md file gives you the database schema, and opencode.jsonc configures the MCP connection and local Ollama models.
Read AGENTS.md and confirm the database context is loaded.

For ChatGPT, Claude Desktop (without MCP), or any other chat interface:

Download CODEBOOK_LLM.md

Open the file, copy its entire contents, and paste it into your first message. This gives the AI the full database schema, query patterns, and connection details in a single paste.

The four skills:

SkillWhat it doesWhen to use
sswr-codebookSchema, field definitions, categorical values, relationships“What fields are in the paper_authors table?”
sswr-query12 SQL query templates for trends, institutions, methodology“Show presentations by year”
sswr-semantic-searchEmbedding similarity search via OpenRouter“Find papers similar to child welfare interventions”
sswr-aboutCitation, provenance, limitations, error reporting“How should I cite this?”

No coding tool? If you’re using a chat interface like Claude Desktop or ChatGPT without file access, download CODEBOOK_LLM.md and paste its contents into your first message. This gives the AI a portable version of the schema and query patterns.

Step 3 — Start Asking Questions

You’re set up. Try these:

“How many presentations were at SSWR in each year from 2005 to 2026?”

“What are the top 20 institutions by first-authored presentations?”

“How has the proportion of qualitative vs. quantitative research changed over time?”

“How has international participation at SSWR changed since 2010?”

“Find presentations about substance use interventions from the last 5 years. Who are the most active researchers?”

“Compare the methodology distribution of University of Michigan vs. Columbia University.”

Prefer files?

The data is also available as SQLite, CSV, and JSON downloads, plus a REST API.
