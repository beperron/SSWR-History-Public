Quick Start

Why AI-first?

There is no scholarly contribution in writing pd.read_csv(). There is no intellectual value in debugging a merge statement or googling the syntax for a grouped bar chart. This is routine analytic code — plumbing that every researcher writes from memory, fixes from Stack Overflow, and forgets by next week.

We are not encouraging researchers to hand off analysis to AI. The opposite: we want to free you from the plumbing so you can invest your time where it actually matters — formulating research questions, reviewing the generated code for correctness, verifying that outcomes make sense, and interpreting what the results mean for the field.

AI handles the code generation. You handle the thinking, the verification, and the scholarship. Your contribution is the question you ask and the interpretation you provide, not the twenty lines of boilerplate that got you there.

Step 1 — Connect to the Database

Give your AI assistant access to the live SSWR database via MCP. Copy this prompt and paste it into Claude Desktop, Claude Code, Cursor, or OpenCode:

Preview
I want to connect to the SSWR Conference History Database via MCP. The server name is “sswr-history” and the URL is https://mcp.supabase.com/mcp?project_ref=jomsksqqcpkbuhwytovo&read_only=true&features=database. Please add this to my MCP configuration, help me restart, and verify it works by listing the available tables.

Already know MCP? The config details are on the Download page.

Step 2 — Install the Skills

The repository includes four modular skills that give the AI specialized capabilities for working with this database. Copy this prompt:

Preview
Please clone the SSWR Conference History Database repository from github.com/beperron/SSWR-History-Public. This repo includes MCP configs (.claude/mcp.json, .cursor/mcp.json) and four skills in .claude/skills/ — sswr-codebook (schema reference), sswr-query (SQL patterns), sswr-semantic-search (embedding search via OpenRouter), and sswr-about (citation and limitations). Read the CLAUDE.md project context and confirm the skills are available.

The skills are:

SkillPurpose
sswr-codebookSchema, field definitions, categorical values, relationships
sswr-query12 SQL query templates for analysis (trends, institutions, methodology, etc.)
sswr-semantic-searchEmbedding-based similarity search via OpenRouter
sswr-aboutCitation, provenance, limitations, error reporting

Using a chat interface without file access? Download CODEBOOK_LLM.md (portable reference) and paste it into your first message.

Step 3 — Start Asking Questions

You’re set up. Try these:

“How many presentations were at SSWR in each year from 2005 to 2026?”

“What are the top 20 institutions by first-authored presentations?”

“How has the proportion of qualitative vs. quantitative research changed over time?”

“How has international participation at SSWR changed since 2010?”

“Find presentations about substance use interventions from the last 5 years. Who are the most active researchers?”

“Compare the methodology distribution of University of Michigan vs. Columbia University.”

Prefer files?

The data is also available as SQLite, CSV, and JSON downloads, plus a REST API.
