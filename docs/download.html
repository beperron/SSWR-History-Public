Download the Data

The SSWR Conference History Database is available in CSV and JSON formats. All files are encoded in UTF-8. See the Codebook for complete field documentation.

Quick Start

Python

import pandas as pd

# Load the data
papers = pd.read_csv('papers.csv')
authors = pd.read_csv('paper_authors.csv')

# Join tables
merged = authors.merge(papers, left_on='paper_id', right_on='id', suffixes=('_author', '_paper'))

# Example: methodology distribution
print(papers['methodology'].value_counts())

# Example: presentations per year
print(papers.groupby('year').size())

R

library(readr)

# Load the data
papers <- read_csv("papers.csv")
authors <- read_csv("paper_authors.csv")

# Join tables
merged <- merge(authors, papers, by.x = "paper_id", by.y = "id")

# Example: methodology distribution
table(papers$methodology)

# Example: top institutions
sort(table(authors$institution_normalized), decreasing = TRUE)[1:20]

Vector Embeddings

Pre-computed vector embeddings are available for all 23,793 presentations via the API. These encode the semantic content of each title and abstract, enabling similarity search, clustering, and other vector-based analyses.

Two models available: text-embedding-3-large (3,072 dims, higher quality) and text-embedding-3-small (1,536 dims, faster/cheaper). Your query must use the exact same model as the embeddings you search against.

ModelDimsRPC FunctionBest For
text-embedding-3-large3,072match_papersPrecision-critical search, clustering
text-embedding-3-small1,536match_papers_smallGeneral similarity search (faster, cheaper)

We recommend OpenRouter for generating query embeddings. OpenRouter provides a unified API that gives you access to both models alongside hundreds of others from OpenAI, Anthropic, Google, Meta, and more — all through a single API key.

Setup (OpenRouter)

  1. Create a free account at openrouter.ai
  2. Generate an API key from your dashboard
  3. Install the OpenAI Python package: pip install openai supabase

Similarity Search

import openai
from supabase import create_client

# --- Your keys (only thing you need to provide) ---
OPENROUTER_KEY = "sk-or-..."   # from openrouter.ai/keys
SUPABASE_KEY   = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImpvbXNrc3FxY3BrYnVod3l0b3ZvIiwicm9sZSI6ImFub24iLCJpYXQiOjE3NDQyMDgwODksImV4cCI6MjA1OTc4NDA4OX0.9Vxp6gB03H0MKxrX1ltSCKCumlT-ba5GszWqAAw1Aqg"

# --- 1. Generate a query embedding via OpenRouter ---
client = openai.OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key=OPENROUTER_KEY,
)

## --- Option A: text-embedding-3-large (higher quality) ---
response = client.embeddings.create(
    model="openai/text-embedding-3-large",
    input="child welfare interventions for foster youth"
)
query_vector = response.data[0].embedding  # 3,072 dims

db = create_client("https://jomsksqqcpkbuhwytovo.supabase.co", SUPABASE_KEY)
result = db.rpc("match_papers", {
    "query_embedding": query_vector,
    "match_threshold": 0.3,
    "match_count": 10
}).execute()

Using OpenAI Directly

If you already have an OpenAI API key, the only difference is the client setup:

client = openai.OpenAI(api_key="sk-...")  # your OpenAI key

# Use the model name without the "openai/" prefix
response = client.embeddings.create(
    model="text-embedding-3-large",       # or "text-embedding-3-small"
    input="your search query here"
)

Explore further: OpenRouter gives you access to models from every major provider through one API. Beyond embeddings, you can use it to run analyses, generate summaries, or build applications on top of this dataset using models from OpenAI, Anthropic, Google, Meta, Mistral, and many others.
