Download the Data

The SSWR Conference History Database is available in CSV and JSON formats. All files are encoded in UTF-8. See the Codebook for complete field documentation.

License: This dataset is released under Creative Commons Attribution 4.0 (CC BY 4.0). You are free to share and adapt the data for any purpose, provided you give appropriate credit. See How to Cite.

CSV Files (Recommended)

Best for use with R, Python (pandas), Stata, SPSS, or spreadsheet software.

23,793 presentations with titles, abstracts, and methodology classifications. All 8 fields from the papers table.

Quick Start

Python

import pandas as pd

# Load the data
papers = pd.read_csv('papers.csv')
authors = pd.read_csv('paper_authors.csv')

# Join tables
merged = authors.merge(papers, left_on='paper_id', right_on='id', suffixes=('_author', '_paper'))

# Example: methodology distribution
print(papers['methodology'].value_counts())

# Example: presentations per year
print(papers.groupby('year').size())

R

library(readr)

# Load the data
papers <- read_csv("papers.csv")
authors <- read_csv("paper_authors.csv")

# Join tables
merged <- merge(authors, papers, by.x = "paper_id", by.y = "id")

# Example: methodology distribution
table(papers$methodology)

# Example: top institutions
sort(table(authors$institution_normalized), decreasing = TRUE)[1:20]

Vector Embeddings

Pre-computed vector embeddings are available for all 23,793 presentations via the API. These encode the semantic content of each presentation’s title and abstract, enabling similarity search, clustering, and other vector-based analyses.

Embedding model: All vectors were generated using OpenAI’s text-embedding-3-large model (3,072 dimensions). Any query embedding must be generated with the same model for meaningful similarity comparisons. The model is available through OpenAI or OpenRouter.

Similarity Search Example

import openai
from supabase import create_client

# 1. Generate a query embedding using the SAME model
client = openai.OpenAI()  # or use OpenRouter
response = client.embeddings.create(
    model="text-embedding-3-large",
    input="child welfare interventions for foster youth"
)
query_vector = response.data[0].embedding

# 2. Search for similar presentations via Supabase RPC
db = create_client(
    "https://jomsksqqcpkbuhwytovo.supabase.co",
    "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."  # anon key
)

# Use the match_papers function or query directly
result = db.rpc("match_papers", {
    "query_embedding": query_vector,
    "match_threshold": 0.7,
    "match_count": 10
}).execute()

AI-Native Access via MCP

The database is available as a Model Context Protocol (MCP) resource. MCP allows AI assistants like Claude to query the database directly through natural conversation — no code required. You ask a question in plain language, and the AI formulates the query, executes it, and interprets the results.

What You Can Do

  • Ask questions like “What percentage of presentations used qualitative methods in 2025?”
  • Run complex analyses: “Which institutions had the most first-authored presentations over the last five years?”
  • Explore trends: “How has international participation changed since 2010?”
  • Cross-reference tables: “What are the most common positions among qualitative researchers?”

Step-by-Step Setup

Prerequisites

  • Node.js (version 18 or later) installed on your system
  • Claude Desktop or Claude Code installed

1. Locate your MCP configuration file

ApplicationConfig File Location
Claude Desktop (macOS)~/Library/Application Support/Claude/claude_desktop_config.json
Claude Desktop (Windows)%APPDATA%\Claude\claude_desktop_config.json
Claude Code~/.claude/settings.json (under mcpServers)

2. Add the SSWR database connection

Open the configuration file and add the following inside the mcpServers object. Create the file or the mcpServers key if it does not exist.

{
  "mcpServers": {
    "sswr-history": {
      "command": "npx",
      "args": [
        "-y",
        "@anthropic-ai/mcp-supabase@latest",
        "--supabase-url",
        "https://jomsksqqcpkbuhwytovo.supabase.co",
        "--supabase-key",
        "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImpvbXNrc3FxY3BrYnVod3l0b3ZvIiwicm9sZSI6ImFub24iLCJpYXQiOjE3NDQyMDgwODksImV4cCI6MjA1OTc4NDA4OX0.9Vxp6gB03H0MKxrX1ltSCKCumlT-ba5GszWqAAw1Aqg",
        "--read-only"
      ]
    }
  }
}

3. Restart the application

Close and reopen Claude Desktop or restart Claude Code. The MCP server will start automatically when the application launches.

4. Verify the connection

In Claude Desktop, look for the MCP server icon (hammer icon) in the bottom-left of the chat window. In Claude Code, the server will appear in the tool list. You can verify by asking:

“List the tables available in the SSWR database and show me how many rows each has.”

5. Start asking questions

Once connected, you can query the database in plain language. Claude will translate your questions into SQL, run them against the live database, and return interpreted results. Example prompts:

  • “Show me the top 20 institutions by number of presentations”
  • “What is the year-over-year growth in mixed methods research?”
  • “How many unique countries are represented in each year?”
  • “Find all presentations by researchers from the University of Michigan in 2024”

Read-only access: The --read-only flag ensures the AI can only query data — it cannot modify, insert, or delete any records. The anon API key provides the same protection at the database level via row-level security policies.
