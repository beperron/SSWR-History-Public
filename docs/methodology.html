Methodology

This page documents how the database was constructed, including data collection, variable extraction, methodology classification, and normalization procedures. For the full description, see the companion paper.

Data Collection

Abstract metadata was compiled from the SSWR Confex conference management system for all available conference years, spanning the 9th Annual Conference (2005) through the 30th Annual Conference (2026). Abstracts prior to 2005 were not publicly available at the time of data collection.

Web Scraping

Data collection used a multi-stage pipeline combining web scraping and language model processing. Custom scraping scripts first retrieved conference program index pages listing all sessions, then systematically accessed individual presentation pages to extract titles, abstracts, and author information. The primary data source was the SSWR Confex archive, accessed via systematic HTTP requests. Web scraping was performed using Crawl4AI, with all data processing implemented in Python (version 3.11).

We compared extracted record counts against conference program indices for each year as an ongoing quality check and believe the database captures the full universe of publicly available abstracts in the Confex system.

Year-Specific Challenges

The HTML structure of conference pages evolved substantially across the 22-year observation period, requiring year-specific parsing logic:

  • 2005–2008: Used a different URL structure and page layout than subsequent years, requiring specialized extraction procedures.
  • 2009–2026: More consistent HTML structure, but still showed variations in underlying formatting across years.
  • 2020–2021: Virtual/hybrid conference formats introduced additional formatting differences.

HTML parsing was performed with BeautifulSoup4 (version 4.12) using the lxml parser backend. For each presentation, titles were extracted from header elements, abstracts from designated content divisions, and author information from affiliation blocks. Iterative validation compared extracted records against manually reviewed samples to ensure parsing accuracy across all conference years.

Inclusion Criteria

All presentation formats containing accompanying abstracts were retained, including oral presentations, poster sessions, symposia, and roundtable discussions. Individual presentations within symposia and roundtable sessions were included, though high-level overview descriptions for these sessions were excluded. Workshops and keynote lectures were excluded as they did not report original scholarship. Presentations lacking abstract text or with abstracts shorter than 50 characters (n = 67) were also excluded.

Variable Extraction & Normalization

To extract structured metadata from unstructured affiliation text, we used the gpt-oss-20b language model deployed locally via llama.cpp. The model parsed raw affiliation strings into standardized fields: author name, academic degrees, position/title, institution name, department, city, state/province, and country/region.

Structured prompts directed the model to extract standardized fields from raw affiliation text, outputting results in JSON format. The prompt included few-shot examples demonstrating the expected output format for various affiliation string patterns common in academic social work contexts. The model was configured with a temperature of 0.1 to minimize output variability. To validate parsing accuracy, we manually reviewed a random sample of 200 parsed records and found that the model correctly extracted institutions in 94% of cases and positions in 91% of cases.

Research Methodology Classification

Each presentation’s methodology was classified based on analysis of its abstract content using a small language model (gpt-oss-20b). The classification prompt instructed the model to analyze each abstract for methodological indicators including data type descriptions, sample characteristics, analytical methods mentioned, and explicit methodology statements.

Validation: A random stratified sample of 60 abstracts (10 per category) was manually classified to calculate inter-rater agreement between human and model classifications. Cohen’s kappa was .83, indicating almost perfect agreement. Accuracy varied by category: quantitative (100%), qualitative (100%), review (90%), mixed methods (80%), and theoretical/other (75%).

Classification Process

Each abstract was classified by the gpt-oss-20b language model, which analyzed the text for methodological indicators including data type descriptions, sample characteristics, analytical methods, and explicit methodology statements. The model’s classification rationale is preserved in the methodology_classification_rationale field.

When the abstract mentioned multiple approaches, classification was based on the dominant methodology unless the abstract explicitly described integrating quantitative and qualitative components, which triggered a mixed_methods classification.

Academic Position Taxonomy

Author positions were extracted from Confex affiliation text by the gpt-oss-20b model, then normalized into 16 standardized categories using keyword-based classification with regular expression pattern matching. Position indicators were searched in priority order to handle records containing multiple position keywords.

Normalization Procedures

Author Names

Author names were normalized using NFKD (Normalization Form Compatibility Decomposition) from Python’s unicodedata module. This process decomposes characters into base forms, removes combining marks, and converts accented characters to ASCII equivalents. Whitespace was collapsed and asterisks marking presenting authors were removed. The resulting normalized names are stored in name_normalized while preserving original casing in the name field.

Institutions

Institution names were normalized through a three-stage matching process: exact matching against canonical forms, substring matching for variants with department prefixes or suffixes, and pattern matching for common naming variations. A language model assisted in creating the mapping table of canonical institution names and their associated variants. The normalization pipeline first cleaned institution strings by removing embedded position titles and geographic suffixes.

Countries/Regions

Country/region names were standardized using a mapping dictionary addressing over 80 name variations. Assignment followed explicit mention in the affiliation text or inference from state/city information. State and province abbreviations within the United States and Canada were expanded to full names.

Entity Resolution

Unique researchers were identified using a graph-based entity resolution system combining fuzzy string similarity (RapidFuzz library), blocking by first initial and last name, and additional factors including matching middle initials, shared institutions, and logical career-stage progression. Name pairs with similarity scores of 90 or higher were connected in a NetworkX graph, and connected components identified author clusters. This process reduced 23,481 raw name variants to 20,779 canonical identities.
