Quick Start

Why AI-first?

This database is designed to be queried through conversation, not code. We are not encouraging researchers to hand off analysis to AI. The opposite: we want to free you from boilerplate so you can focus entirely on thinking about your research questions, interpreting results, and understanding the data.

Writing pd.read_csv() or library(readr) is not a scholarly contribution. It’s plumbing. The intellectual work is deciding what to ask, how to interpret, and what it means. AI handles the plumbing. You do the thinking.

Step 1 — Connect to the Database

Add the SSWR database to your AI tool’s MCP configuration. This gives your AI assistant direct, read-only access to the live database.

Option A: Let your AI set it up

Copy this prompt and paste it into Claude Desktop, Claude Code, Cursor, or OpenCode:

Preview what gets copied

I want to connect to the SSWR Conference History Database via MCP. Please add this to my MCP config and help me restart:

{"mcpServers":{"sswr-history":{"url":"https://mcp.supabase.com/mcp?project_ref=jomsksqqcpkbuhwytovo&read_only=true&features=database"}}}

Then verify by listing the available tables.

Option B: Add it yourself

Add this to your MCP config file (no API key or Node.js needed):

{
  "mcpServers": {
    "sswr-history": {
      "url": "https://mcp.supabase.com/mcp?project_ref=jomsksqqcpkbuhwytovo&read_only=true&features=database"
    }
  }
}

Restart your application. The database is now connected.

Step 2 — Install the Skill Files

Skill files give the AI complete context about the database schema, field definitions, query patterns, and data quality notes. Without them, the AI can still query the database but may not know the best join patterns or categorical values.

git clone https://github.com/beperron/SSWR-History-Public.git
cd SSWR-History-Public

The repository contains three skill files that are read automatically:

FileForWhat it does
CLAUDE.mdClaude CodeFull schema, query patterns, data quality notes
AGENTS.mdOpenCodeSame content, for OpenCode’s agent system
CODEBOOK_LLM.mdAny LLMCompact reference — paste into any AI chat window

If you’re using Claude Desktop (not Claude Code), paste the contents of CODEBOOK_LLM.md into your first message to give Claude the full dataset context.

Step 3 — Start Asking Questions

You now have everything set up. Here are real questions you can ask right now:

Explore the data

“How many presentations were at SSWR in each year from 2005 to 2026? Show the trend.”

“What are the top 20 institutions by first-authored presentations?”

Analyze trends

“How has the proportion of qualitative vs. quantitative research changed over time? Show percentages by year.”

“What percentage of first authors are doctoral students? Has this changed?”

Answer research questions

“How has international participation at SSWR changed since 2010? Which countries are growing fastest?”

“Compare the methodology distribution of presentations from University of Michigan vs. Columbia University.”

Discover connections

“Find presentations about substance use interventions from the last 5 years. Who are the most active researchers in this area?”

Traditional Access

If you prefer working with files directly, the data is available as CSV, JSON, and SQLite downloads. A REST API is also available for programmatic access.
